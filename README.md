# Помогать проСТО — Корпоративная волонтерская платформа Столото (MVP)

Единая цифровая платформа для организации, сопровождения и учёта волонтёрской деятельности сотрудников Столото. Решение разработано на современном стеке **Node.js (TypeScript) + Express + React** с возможностью быстрого бесшовного развертывания.

Основная цель проекта — обеспечение абсолютной прозрачности взаимодействия между корпоративными волонтёрами, благотворительными фондами-партнёрами и кураторами (администрацией) Столото, автоматизация учёта часов участия (ESG-показатели) и геймификация признания заслуг.

---

## 🛠 Технологический стек

*   **Backend:** Node.js, Express, TSX (серверный TypeScript без лишней компиляционной нагрузки), fs-based локальная СУБД с транзакционной сериализацией (симуляция PostgreSQL для автономности MVP).
*   **Frontend:** React 19 (TypeScript), Tailwind CSS v4, Lucide Icons, Modern Motion.
*   **Сборка:** Vite 6, Esbuild (bundle-сборка серверного слоя для Cloud Run).

## 👥 Роли и Права Доступа

### 1. Корпоративный Волонтёр (Сотрудник Столото)
*   **Авторизация:** По корпоративному адресу электронной почты (`*@stoloto.ru`).
*   **Личный кабинет (ЛК):** Интерактивный прогресс-бар часов, текущий ранг (Новичок ➔ Активный ➔ Лидер), коллекция наград за верифицированные категории помощи.
*   **Лента проектов:** Поиск и фильтрация по формату (онлайн/офлайн), длительности, типу участия (обычные / Pro-Bono) и городам присутствия.
*   **Действия:** Подача заявок на участие (отклик с проверкой лимита мест), отзыв поданного отклика в статусе ожидания.
*   **Уведомления:** Лента системных алертов о подтверждениях от фондов и начислении часов.

### 2. Благотворительный Фонд (Партнер платформы)
*   **Авторизация:** По зарегистрированной почте организации.
*   **Пре-модерация:** Обязательная юридическая проверка ИНН/ОГРН администратором. До одобрения создание заданий заблокировано.
*   **Управление проектами:** Составление заданий (сбор денежных средств запрещен!), отправка на пре-модерацию Столото, досрочное закрытие.
*   **Управление кандидатами:** Принятие/отклонение откликов волонтёров, отметка личного участия сотрудников с фиксацией обратной связи.
*   **Аналитика:** Скачивание CSV-отчёта об активности фонда.

### 3. Администратор (ESG-координатор Столото)
*   **Модерация фондов:** Утверждение или отклонение документов регистрирующихся фондов.
*   **Модерация заданий:** Проверка условий волонтёрских акций, утверждение публикации в ленту.
*   **Часовой фонд:** Ручное зачисление волонтёрских часов по результатам подтверждений от фондов.
*   **Аналитика:** Интерактивные чарты активности департаментов и популярности категорий, скачивание общего CSV-реестра волонтёров.

## 📊 Таблица статусов и Статусная логика

### Модерация фонда (FundStatus)
| Статус | Код | Описание |
| :--- | :--- | :--- |
| **Ожидает проверки** | `PENDING` | Фонд зарегистрирован, документы загружены. Инструменты создания дел заблокированы. |
| **Одобрен** | `APPROVED` | Проверен администратором Столото, разрешена полная работа. |
| **Отклонен** | `REJECTED` | Заявка не подтверждена. Направлен комментарий юристов. |

### Контур задания (TaskStatus)
```
[Черновик (DRAFT)] ➔ [Модерация (PENDING_MODERATION)] ➔ [Опубликовано (PUBLISHED)] ➔ [Выполнено (COMPLETED)]
                                                    ➔ [Отклонено (REJECTED)]
```

### Статус заявки волонтёра (ApplicationStatus)
```
[Ожидание фонда (PENDING)] ➔ [Принят (ACCEPTED)] ➔ [Подтверждено фондом (COMPLETED) + Начислены часы]
                           ➔ [Отклонено/Отозвано (REJECTED)]
```

---

## ⏱ Логика начисления волонтёрских часов

Во избежание накрутки и фальсификации, часы **никогда не начисляются автоматически**:
1.  Волонтёр успешно завершает участие в проекте.
2.  Фонд подтверждает участие кандидата в ЛК и вносит отзывы в поле `Описание вклада`.
3.  Администратор Столото получает уведомление о закрытом слоте, проверяет отчёт и кликает **«Зачислить часы»** (со спец-правом редактирования итогового времени).
4.  Часы переносятся на баланс волонтёра, продвигая его ранг и открывая бейджи (по категориям закрытых заданий).

## 🚀 10 Сценариев последовательной проверки Жюри

В правом нижнем углу интерфейса встроена **Интерактивная демо-панель проверки**. При нажатии на любой из 10 шагов сценария, система автоматически авторизует требуемую роль и переключает интерфейс на нужный раздел:

1.  **Регистрация волонтёра:** Волонтёр Пётр (`petrov@stoloto.ru`) регистрируется, получает префикс Новичка и видит ЛК с заблокированными серыми наградами.
2.  **Регистрация фонда:** БФ «Подари Свет» (`fond_help@hope.ru`) подаёт соглашение с ИНН и ОГРН. Получает статус `PENDING`. Запрет создания заданий активен.
3.  **Модерация фонда:** Входим как `admin@stoloto.ru`, одобряем фонд во вкладке «Модерация фондов».
4.  **Создание задания:** Авторизуемся одобренным фондом (`fond@nature.ru`), пишем параметры задачи «Дизайн благотворительного баннера», маркер Pro-Bono, дедлайн, и кликаем «Отправить на модерацию».
5.  **Публикация задания:** Будучи Админом, заходим в раздел «Модерация заданий», проверяем материалы и подтверждаем публикацию.
6.  **Фильтрация и поиск:** Заходим волонтёром Иваном (`volunteer@stoloto.ru`). Используем фильтр по тегу «Pro-bono» и городу, находим созданную карточку проекта.
7.  **Отклик волонтёра:** Нажимаем на карточку, вызываем модальное окно, знакомимся с условиями и нажимаем кнопку «Откликнуться». Места резервируются в реальном времени.
8.  **Принятие и подтверждение:** Входим фондом (`fond@nature.ru`), заходим во вкладку «Заявки», переводим Ивана из статуса `Одобрить` в финальное участие кнопкой «Подтвердить фактическое участие» с текстом благодарности.
9.  **Начисление часов:** Админ открывает вкладку «Начисление часов», видит согласованный фонд-аккаунт, редактирует/подтверждает 8 часов и нажимает «Зачислить». Волонтёр Иван моментально получает Push-сигнал, у него растет прогресс-бар в ЛК и загорается цветной бейдж «IT-PRO специалист».
10. **Скачивание отчета:** Админ заходит в раздел Аналитики и скачивает общий сквозной отчет волонтёров Столото в кодировке UTF-8 с BOM (для считывания кириллицы в Excel).

## 📦 Инструкция по запуску

Для локального запуска выполните:

```bash
# 1. Установка всех пакетов
npm install

# 2. Сборка фронтенда и бэкенда в продуктивный дистрибутив
npm run build

# 3. Запуск веб-приложения на порту 3000
npm run start
```

Платформа запустится на порту `3000`. Локальная база сохраняется в файле `database.json` в корневой директории и автоматически инициализируется демонстрационными пресетами при его отсутствии.
